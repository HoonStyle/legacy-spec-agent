## System Purpose
hookify turns user-authored rule files into Claude Code hooks that **warn on or block** tool calls, prompts, and stop events by pattern-matching the conversation/tool payload. A hook entrypoint reads the event payload from stdin, loads the rules that apply to that event, evaluates them, and prints a JSON decision — never crashing the operation. (`hooks/pretooluse.py:39`, `core/config_loader.py:198`, `core/rule_engine.py:35`)

## Modules

### Rule Engine — `core/rule_engine.py`
- **Responsibility**: Evaluate Rule objects against a hook's `input_data` and return an event-appropriate response (block / warn / allow).  `core/rule_engine.py:35`
- **Business rules**:
  - A matched rule is **blocking only when `rule.action == 'block'`**; every other action value is treated as a warning.  `core/rule_engine.py:55`
  - **Precedence — blocking wins**: if any blocking rule matched, only the blocking branch runs and warnings are dropped.  `core/rule_engine.py:61`
  - Blocking messages are combined, each formatted `**[{name}]**\n{message}`, joined by two newlines.  `core/rule_engine.py:62`
  - Event-shaped block response: `Stop` → `{decision:'block', reason, systemMessage}`; `PreToolUse`/`PostToolUse` → `hookSpecificOutput.permissionDecision:'deny'` + `systemMessage`; any other event → `{systemMessage}` only.  `core/rule_engine.py:66`
  - Warnings-only → `{systemMessage}` and the operation is allowed; no matches → `{}` (allow).  `core/rule_engine.py:87`
  - A rule with **no conditions never matches**; conditions are **AND-combined** (all must match).  `core/rule_engine.py:117`
  - Tool matcher: `'*'` matches any tool; otherwise the matcher is split on `|` and `tool_name` must be an **exact** member (OR match — not regex/substring).  `core/rule_engine.py:137`
  - Supported operators: `regex_match, contains, equals, not_contains, starts_with, ends_with`; any unknown operator → no match.  `core/rule_engine.py:166`
  - A `None` extracted field value fails the condition immediately.  `core/rule_engine.py:159`
  - Field extraction is tool-aware: `Bash`→`command`; `Write/Edit`→`content` (falls back to `new_string`); `MultiEdit`→ all edits' `new_string` joined by spaces.  `core/rule_engine.py:231`
- **Inputs / Outputs**: in = `rules: List[Rule]`, `input_data: dict`; out = response dict per shapes above.  `core/rule_engine.py:35`
- **Side effects**:
  - Field `transcript` opens and reads `input_data['transcript_path']`; FileNotFound/Permission/IO/OS/UnicodeDecode errors are caught, warned to stderr, and yield `''`.  `core/rule_engine.py:207`
  - `_regex_match` uses a cached compiled regex and `re.search` (not `fullmatch`); invalid regex is caught and yields `False`.  `core/rule_engine.py:269`
- **Constraints**:
  - **All regex matching is case-insensitive** — `compile_regex` hardcodes `re.IGNORECASE` (memoized, max 128 patterns).  `core/rule_engine.py:14`
  - Non-regex operators are **case-sensitive** plain-string comparisons.  `core/rule_engine.py:169`

### Config Loader — `core/config_loader.py`
- **Responsibility**: Discover, read, and parse `.claude/hookify.*.local.md` rule files (YAML frontmatter + markdown body) into `Condition`/`Rule` dataclasses, filtered by event and enabled state.  `core/config_loader.py:198`
- **Business rules**:
  - `Rule.from_dict` prefers a new-style `conditions` list; **legacy `pattern`** is converted to a single `regex_match` condition **only when no `conditions` exist**, with field inferred from event (`bash`→`command`, `file`→`new_text`, else→`content`).  `core/config_loader.py:57`
  - `Rule` defaults: `enabled=True`, `event='all'`, `action='warn'`, `conditions=[]`, `tool_matcher=None`, `message=''` (stripped).  `core/config_loader.py:32`
  - `load_rules(event)` globs `.claude/hookify.*.local.md`, returns **enabled** rules where `rule.event == 'all'` or `rule.event == event`.  `core/config_loader.py:221`
  - `extract_frontmatter` returns `({}, content)` unless content starts with `---` and splits into ≥3 `---` parts.  `core/config_loader.py:87`
  - Frontmatter is parsed by a **bespoke line-based parser (not PyYAML)**: skips blanks and `#` comments; strips surrounding quotes; coerces `true`/`false` to bool; inline list-dict items require both `:` and `,`.  `core/config_loader.py:98`
- **Side effects**: reads rule files from disk; per-file IO/parse/unexpected errors are caught, warned to stderr, and skipped (loop continues).  `core/config_loader.py:228`
- **Constraints**:
  - Discovery is **CWD-relative** (`os.path.join('.claude', ...)`) — rules outside the process working directory's `.claude` are not found.  `core/config_loader.py:210`
  - A file must begin with `---` frontmatter; empty-parsed frontmatter → invalid (`None`).  `core/config_loader.py:256`
  - `load_rule_file` returns **one Rule per file** (multiple rules per file unsupported).  `core/config_loader.py:250`

### Hook Entrypoints — `hooks/{pretooluse,posttooluse,stop,userpromptsubmit}.py`
- **Responsibility**: Per-event script entrypoints. Read the event payload from stdin, resolve the event label, load matching rules, evaluate them, print a JSON decision.  `hooks/pretooluse.py:39`
- **Business rules**:
  - **Event resolution differs by hook**: `pretooluse`/`posttooluse` map `Bash`→`bash`, `Edit|Write|MultiEdit`→`file`, else no event filter (`None`).  `hooks/pretooluse.py:46` · `hooks/posttooluse.py:40`
  - `stop` always loads the **fixed event `'stop'`**; `userpromptsubmit` always loads the **fixed event `'prompt'`**.  `hooks/stop.py:37` · `hooks/userpromptsubmit.py:37`
  - **Fail-open guarantee**: on any exception, print `{"systemMessage": "Hookify error: ..."}` and **always `sys.exit(0)`** in `finally` — a hook error never blocks the operation.  `hooks/pretooluse.py:63`
  - Result JSON is **always printed to stdout, even when empty**.  `hooks/pretooluse.py:59`
- **Side effects / external calls**: `load_rules(event=event)` then `RuleEngine().evaluate_rules(rules, input_data)`.  `hooks/pretooluse.py:52`
- **Constraints**: require `CLAUDE_PLUGIN_ROOT` to inject `sys.path` so the `hookify` package resolves; import failure prints an error and exits 0.  `hooks/pretooluse.py:14`

## Data & Control Flow
Claude Code fires a hook → the matching entrypoint reads stdin JSON → resolves an event label → `config_loader.load_rules(event)` reads `.claude/hookify.*.local.md` from the CWD → `rule_engine.evaluate_rules` matches rules (blocking beats warning) and reads `transcript_path` when a rule inspects the transcript → the entrypoint prints the JSON decision and exits 0. See `ARCHITECTURE.md` for the graph.

## Constraints & Assumptions (system-wide)
- Rules live only under the **process CWD's `.claude/`**; running from another directory silently loads no rules.  `core/config_loader.py:210`
- Regex is always case-insensitive; substring/equality operators are not.  `core/rule_engine.py:14` · `core/rule_engine.py:169`
- Hooks are **fail-open** by contract — misconfiguration weakens enforcement rather than breaking the session.  `hooks/pretooluse.py:70`
