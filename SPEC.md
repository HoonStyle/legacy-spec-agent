# Legacy Spec Agent — 제품·기술 명세서 (v0.1)

> 문서 없는 레거시 코드에서 "이 시스템이 실제로 무엇을 하는지"를 근거(citation) 있는 명세서로 역생성하고,
> 이후 코드 변경이 그 명세와 어긋나는 지점(drift)을 감사로그로 추적하는 멀티에이전트 시스템.

- **작성일**: 2026-07-15
- **버전**: v0.1 (최초 설계 문서. 이후의 구현 방향은 `CONNECTOR_DESIGN.md`에서 확정)
- **워킹 네임**: `Legacy Spec Agent` (제품명 변경 가능)

---

## 0. 한 줄 정의

> **"명세서를 대조하는 도구가 아니라, 없는 명세서를 코드에서 뽑아내는 기계."**
> 코드는 항상 존재하고 문서는 거의 없다 — 그 비대칭을 입력 조건으로 삼는다.

---

## 1. 문제 정의 (Problem)

- 현실의 레거시 시스템 대다수는 **스펙·설계 문서가 없거나 코드와 어긋나 있다.** 개발자들은 문서를 잘 쓰지 않는다.
- 그래서 "코드 ↔ 스펙 대조" 류 도구는 **입력(스펙)이 존재한다는 전제 자체가 깨진다.**
- 반면 **코드는 언제나 있다.** 진짜 병목은 "문서가 없어서 이 시스템이 뭘 하는지 파악하는 데 드는 비용"이다.
- 리팩토링·인수인계·규제 대응·유지보수 착수 시점마다 이 파악 비용이 반복 발생한다.

### 이 프로젝트가 뒤집는 지점
"문서가 없다"는 문제를 **입력 조건**으로 재정의한다. 문서를 요구하지 않고, 코드에서 명세를 역생성한다.

---

## 2. 타겟 사용자 / 도메인

- **1차 사용자**: 문서 없는 레거시를 인수받아 리팩토링·유지보수를 시작해야 하는 개발 리드 / TPM
- **도메인 핏(포지셔닝)**: 제조 DX / Industrial AI / 엔터프라이즈 레거시. 규제 도메인(헬스케어)에서는 "감사 추적" 축이 특히 값이 큼
- **비타겟**: B2C / 핀테크 / 커머스 (프로젝트 성격상 해당 없음)

---

## 3. 핵심 가치 제안 (Value Proposition)

| # | 가치 | 누구에게 |
|---|------|----------|
| 1 | 문서 0 → "이 시스템 실제 동작" 명세서 자동 생성 | 인수인계 받는 개발 리드 |
| 2 | 모든 명세 항목에 **코드 근거(citation)** → 할루시네이션 방어 | 검증이 필요한 규제/품질 담당 |
| 3 | 코드 변경이 역생성 명세와 어긋나는 지점 **drift 감사로그** | 유지보수·거버넌스 담당 |

---

## 4. 기능 요구사항 (Functional Requirements)

### MVP (Must)
- **F1. 코드베이스 인제스트**: 로컬 디렉토리(단일 언어 우선, 예: Python) 파싱 → 파일/함수/클래스 단위 청크화
- **F2. 코드 RAG 인덱싱**: 청크 임베딩 + 로컬 벡터스토어 저장, 의존성(호출 관계) 메타 부착
- **F3. 명세 역생성**: 청크에서 "무엇을/왜 하는가"(기능·비즈니스 규칙·입출력·제약)를 구조화 추출
- **F4. 근거 부착**: 모든 명세 항목에 `파일:라인` citation 필수. 근거 없는 항목은 "미검증"으로 격리
- **F5. 산출물 출력**: `SPEC.md`(역생성 명세) + 의존성 맵(Mermaid) 생성

### v1 (Should)
- **F6. Drift 추적**: 이전 인덱스 스냅샷 vs 현재 코드 diff → 명세와 어긋난 항목 플래그
- **F7. 감사로그**: 각 명세 항목의 생성/변경 근거를 `audit_log.jsonl`로 append-only 기록
- **F8. 최소 웹 UI**: 레포 경로 입력 → 리포트 + 의존성 맵 열람 (데모용)

### 비범위 (Won't — 최소 이번 사이클)
- 다중 언어 동시 지원, 자동 코드 수정/PR 생성, 실시간 CI 통합, 대규모(수십만 LOC) 스케일 튜닝

## 5. 멀티에이전트 아키텍처 (LangGraph)

상태 그래프 기반. 단일 프롬프트 컨텍스트 한계를 넘고, 단계별 독립 검증을 강제해 할루시네이션을 억제한다.

```
[Ingestor] → [Spec Extractor] → [Architect] → [Critic/Validator] → [Drift Tracker]
     │              │                │                 │                   │
   파싱·청크    기능·규칙 역생성   모듈 경계·        근거 대조·          스냅샷 diff·
   RAG 인덱스                     데이터흐름 맵     미검증 격리          감사로그
```

| 에이전트 | 역할 | 핵심 산출 |
|---|---|---|
| **Ingestor** | 코드 파싱(AST), 청크화, 임베딩·인덱싱, 의존성 메타 | 벡터 인덱스 + 의존성 그래프 |
| **Spec Extractor** | 청크별 기능/비즈니스 규칙/입출력/제약 역생성 | 명세 항목(초안) |
| **Architect** | 모듈 경계·서비스 바운더리·데이터 흐름 종합 | 아키텍처 맵(Mermaid) |
| **Critic/Validator** | 각 항목이 코드 근거로 뒷받침되는지 검증, 근거 없으면 "미검증" 격리 | 검증된 명세 + 미검증 목록 |
| **Drift Tracker** | 스냅샷 diff → 명세 어긋남 탐지, 감사로그 append | Drift 리포트 + audit_log |

> 설계 원칙: **근거 없는 서술은 명세에 올리지 않는다.** (Critic 게이트) — "Fact First" 규율을 제품 로직으로 구현.

## 6. 기술 스택 (Tech Stack)

- **Language**: Python 3.11+
- **Orchestration**: LangGraph (상태 그래프 / 멀티에이전트)
- **LLM**: Claude (추론: `claude-opus-4-8` / 대량 처리: `claude-sonnet-5`)
- **코드 파싱**: tree-sitter (언어별 AST) — MVP는 Python 우선
- **RAG**: 로컬 임베딩 + 벡터스토어(Chroma 등). AnythingLLM 류 스택과 개념적으로 동일 계열
- **다이어그램**: Mermaid
- **UI(데모)**: Streamlit(빠른 데모) 또는 최소 정적 리포트

> v0.1 이후 결정: 오케스트레이션·RAG·UI 스택은 `CONNECTOR_DESIGN.md` §5에서 재해석됨 (LangGraph→스킬 fan-out, Chroma→tree-sitter 심볼 인덱스, Streamlit→정적 산출물).

## 7. 데이터 흐름 (Pipeline)

```
레거시 코드 디렉토리
   │  (Ingestor: AST 파싱 → 청크 → 임베딩)
   ▼
벡터 인덱스 + 의존성 그래프
   │  (Spec Extractor: 청크→명세 초안 / Architect: 경계·흐름)
   ▼
명세 초안 + 아키텍처 맵
   │  (Critic: 코드 근거 대조, 미검증 격리)
   ▼
검증된 SPEC.md  ── 이후 코드 변경 ──▶ (Drift Tracker: diff) ──▶ DRIFT_REPORT.md + audit_log.jsonl
```

---

## 8. 산출물 포맷 (Deliverables)

- **`SPEC.md`**: 섹션별 기능·비즈니스 규칙·입출력·제약. **모든 항목에 `파일:라인` citation.** 근거 없는 항목은 별도 "미검증" 섹션으로 분리
- **`ARCHITECTURE.md`**: Mermaid 의존성/데이터 흐름 맵
- **`DRIFT_REPORT.md`**: 변경 감지 시, 명세와 어긋난 항목 목록 + 근거 diff
- **`audit_log.jsonl`**: append-only. 각 라인 = {항목ID, 액션(created/updated/flagged), 근거(파일:라인), 타임스탬프}

---

## 9. 데모 시나리오

1. 문서 부실한 중간 규모 오픈소스/레거시 레포를 입력
2. 실행 → 몇 분 내 `SPEC.md` + 의존성 맵 생성, 각 규칙에 코드 근거가 달림
3. 코드 한 곳을 의도적으로 변경 → 재실행 → **Drift 리포트에 "명세와 어긋남" 자동 플래그 + 감사로그 기록**
