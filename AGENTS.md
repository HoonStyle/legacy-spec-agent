# Repository session startup

At the start of every work session in this repository, read `IMPLEMENTATION_ROADMAP.md` before planning or editing. Treat its numbered order, current status, exit gates, and commit policy as the default execution order unless the user explicitly changes priorities.

Read `TOKEN_USAGE.md` before changing token metrics or benchmarks. Read `TOOLCHAIN_EXECUTION_PLAN.md` before changing SDK assessment, consent, download, extraction, installation, or execution behavior.

Keep work split into independently buildable functional commits as required by the roadmap. Do not mark a priority complete until its documented exit gate is satisfied; when only CI can satisfy the final gate, state that explicitly.
