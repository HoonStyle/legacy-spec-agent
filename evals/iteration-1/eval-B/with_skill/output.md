## System Purpose
A Bun script that housekeeps GitHub issues for a repo in two passes: (1) label inactive issues `stale`, and (2) close issues whose lifecycle label has aged past its per-label timeout. It talks to the GitHub REST API directly, supports a `--dry-run` preview mode, and derives all labels/timeouts/messages from the shared `lifecycle` table. `sweep.ts:165-166`, `sweep.ts:3`

- **Responsibility**: Label long-inactive open issues `stale`.  `sweep.ts:45-90`

  - Cutoff date = now − staleDays; issues with `updated_at` older than cutoff are candidates.  `sweep.ts:47-48`, `sweep.ts:65`
  - Scans open issues sorted by `updated` ascending, `per_page=100`, pages 1–10.  `sweep.ts:54-57`
  - Skips pull requests, locked issues, and any issue with ≥1 assignee.  `sweep.ts:61-63`
  - **Early full stop**: on the first issue with `updated_at` newer than cutoff, the entire pass returns (relies on ascending sort — everything after is newer).  `sweep.ts:65-66`
  - Skips issues already carrying label `stale` or `autoclose`.  `sweep.ts:68-71`
  - Skips issues whose `+1` reaction count ≥ `STALE_UPVOTE_THRESHOLD` (10) — upvoted issues are protected.  `sweep.ts:73-74` (value: `issue-lifecycle.ts:38`)
  - Non-dry-run action: `POST .../labels` adding `["stale"]`.  `sweep.ts:82`
  - Dry-run: logs `would label stale (<age>d inactive)` where age is floor of ms/86400000.  `sweep.ts:78-80`
- **Inputs / Outputs**: In — `owner`, `repo`. Out — count of issues labeled.  `sweep.ts:45`, `sweep.ts:89`

### Expiry-closing pass — `closeExpired()`
- **Responsibility**: Close issues whose lifecycle label has aged past that label's timeout.  `sweep.ts:92-154`
- **Business rules**:
  - Iterates **every** lifecycle entry, so it closes per label/timeout: `invalid`/3d, `needs-repro`/7d, `needs-info`/7d, `stale`/14d, `autoclose`/14d.  `sweep.ts:95-97` (table: `issue-lifecycle.ts:3-34`)
  - Per label, scans open issues filtered `labels=<label>`, sorted `updated` ascending, `per_page=100`, pages 1–10.  `sweep.ts:100-103`
  - Skips pull requests and locked issues.  `sweep.ts:107-108`
  - Skips issues with `+1` ≥ 10 (upvote protection).  `sweep.ts:110-111` (value: `issue-lifecycle.ts:38`)
  - Age is measured from **when the label was applied**, not `updated_at`: reads the issue events, filters `event === "labeled"` matching the label, and takes the most recent via `.pop()`.  `sweep.ts:115-120`
  - Skips if no matching label event was found, or the label was applied more recently than cutoff (not old enough).  `sweep.ts:122`
  - **Human-activity safety net**: fetches comments `since=<labeledAt>`; if any comment author `type !== "Bot"`, skips and logs.  `sweep.ts:127-138`
  - Non-dry-run action is two calls in order: post `CLOSE_MESSAGE(reason)` comment, then PATCH the issue to `state: "closed"`, `state_reason: "not_planned"`.  `sweep.ts:144-145`
  - `CLOSE_MESSAGE` text embeds the label's `reason` and a link to the new-issue chooser at `github.com/anthropics/claude-code/issues/new/choose`.  `sweep.ts:7`, `sweep.ts:10-11`
  - Dry-run: logs `would close (<label>, <age>d old)`.  `sweep.ts:140-142`
- **Inputs / Outputs**: In — `owner`, `repo`. Out — count of issues closed.  `sweep.ts:92`, `sweep.ts:153`

## Data & Control Flow
`sweep.ts` imports the `lifecycle` table and `STALE_UPVOTE_THRESHOLD` from `issue-lifecycle.ts` (the single source of truth for labels/timeouts/messages). The entry block validates env, then calls `markStale` (label pass) followed by `closeExpired` (close pass); both call the shared `githubRequest` client for all GitHub I/O. `markStale` reads the open-issue list; `closeExpired` additionally reads each candidate's events and comments before acting. All mutations (labeling, commenting, closing) are gated behind `!DRY_RUN`.  `sweep.ts:3`, `sweep.ts:165-166`

## Constraints & Assumptions
- The `lifecycle` table **must** contain an entry with `label === "stale"`; a non-null assertion (`!`) will throw at runtime if it does not.  `sweep.ts:46`
- Each query is capped at 10 pages × 100 = **1000 issues**; beyond that, issues are silently not processed in that run.  `sweep.ts:54`, `sweep.ts:100`
- `markStale` early-return correctness depends on the `direction=asc` sort; changing the sort direction would break the stop condition.  `sweep.ts:56`, `sweep.ts:65-66`
- `closeExpired` reads only the first 100 events (`events?per_page=100`, no pagination); on issues with >100 events the `labeled` event could be missed.  `sweep.ts:115`
- `.pop()` on the filtered/mapped events assumes GitHub returns events oldest→newest so the last is the most recent label application.  `sweep.ts:117-120`
- The human-activity check reads only the first 100 comments since the label date (`per_page=100`, no pagination).  `sweep.ts:127-129`
- Both env vars for owner/repo and the `GITHUB_TOKEN` must be present or the script throws before/at first request.  `sweep.ts:20-21`, `sweep.ts:158-161`
