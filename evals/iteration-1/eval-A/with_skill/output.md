## System Purpose
Git-derived helpers that let the security-guidance plugin decide **which files an AI turn actually changed**, so a Stop-time security review only inspects Claude-authored edits and not the user's pre-existing work-in-progress. It manages a per-session git "baseline" snapshot, tracks touched paths with TTL/dedup state, and computes a "v2 review set" by intersecting *dirty-vs-HEAD* files with *changed-since-baseline* files (`diffstate.py:353`, `diffstate.py:426`). It also maintains a repo-local append-only log of already-reviewed commit SHAs so commit/push reviews can dedup (`diffstate.py:242`, `diffstate.py:267`). The module is a readability extraction from `security_reminder_hook.py` and holds no entry point of its own — it is a library of functions called by the hook (`diffstate.py:1`).

## Data & Control Flow
Per session, a caller (the security hook) captures a baseline SHA via `capture_git_baseline` and records paths Claude touches via `record_touched_path`; both persist through `with_locked_state`. At Stop time, `consume_stop_state` atomically snapshots that state and clears `touched_paths`. `compute_v2_review_set` then reads live git status/HEAD and intersects the currently-dirty set with the changed-since-baseline set to produce the absolute review paths and a `diff_base` the caller feeds to `git diff`. If the Stop review aborts transiently, `restore_unreviewed_stop_state` puts the consumed paths back. Separately, reviewed commit SHAs are appended to a `.git/`-local capped log for commit/push dedup. See ARCHITECTURE.md for the module/dependency diagram (external: `git` CLI, `gitutil`, `session_state`, `_base`).
